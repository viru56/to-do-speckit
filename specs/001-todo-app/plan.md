# Implementation Plan: Simple Todo App

**Branch**: `001-todo-app` | **Date**: 2026-03-09 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/specs/001-todo-app/spec.md`

## Summary

Build a full-stack Todo application as a **pnpm + Turborepo monorepo**. The backend is a Fastify REST API backed by PostgreSQL (via Prisma); the frontend is React with Zustand state management and TanStack Router v1. A shared `@todo-app/types` package eliminates type duplication between apps. JWT-based authentication with per-user todo isolation is included as a deliberate scope expansion requested during planning.

## Scope Expansion Notice

> **The original spec explicitly excludes user accounts and authentication.** This plan incorporates them as an architectural upgrade at the user's explicit request during the planning phase. The following spec items are superseded:
>
> - *Spec exclusion*: "User accounts, authentication, and authorization" → **Included in this plan**
> - *Spec exclusion*: "Multi-user or collaborative todo lists" → **Included (per-user isolation, not collaboration)**
> - *Spec assumption*: "A single shared list is acceptable for v1" → **Superseded: todos are now user-scoped**
>
> All other spec requirements remain in force.

## Technical Context

**Language/Version**: Node.js 20 LTS, TypeScript 5.4 (full-stack)  
**Package Manager**: pnpm 9 (workspaces)  
**Build Orchestration**: Turborepo 2  
**Primary Dependencies**: Fastify 5, Prisma 5, @fastify/jwt v10, bcryptjs, React 19, Zustand 5, TanStack Router v1, Vite 5, Zod  
**Storage**: PostgreSQL 16 (via Prisma ORM in `apps/backend`)  
**Testing**: Vitest (full-stack — frontend components + backend integration)  
**Target Platform**: Web browser (desktop + mobile) + Node.js server  
**Project Type**: Full-stack monorepo web application  
**Performance Goals**: All CRUD operations complete in UI in under 1 second; auth endpoints under 500ms  
**Constraints**: Access-token-only JWT (15-minute expiry); bcryptjs password hashing; todos are per-user  
**Scale/Scope**: Single-user personal app; no multi-tenancy or collaboration in v1

**Structure Decision**: pnpm workspaces + Turborepo monorepo with `apps/` and `packages/` tiers. Applications (backend, frontend) live in `apps/`; reusable shared code lives in `packages/`. The only shared package in v1 is `@todo-app/types`, which provides a single source of truth for TypeScript types used by both the backend and frontend. Internal package references use the `workspace:*` protocol.

The `^build` dependency ensures `@todo-app/types` is built before the apps that depend on it.
